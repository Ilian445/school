## Basic commands
- `docker ps` - Показать контейнеры, которые работают (`-a` - Показать все контейнеры, включая неработающие)
- `docker stop [id/name container]` - Остановить контейнер
- `docker rm [container name/id]` - Удалть контейнер
- `docker images` - Показать образы которые есть на хосте
- `docker rmi [image id/name]` - Удалить образ
- `docker pull` - Скачать образ ()
- `docker run [name/id]` - Запустить образ/контейнер (`-d` - Запустить в фоне (без привязки к консоли), `-it` - запустить в консоли (с привязкой), `--name [string]` -  назначить имя контейнера)
- `docker exec [id/name container] [command]` - Выполнить команду внутри контейнера
- `docker attach [container name/id]` - Прикрепить вывод контейнера к консоли
- `docker run [image name]:[tag e.x. - version]` - Запустить образ определённой версии
- `docker inspect [container name/id]` - Показать подробную информацию о контейнере (В JSON формате)
- `docker logs[container name/id]` - Статус контейнера(показывает логи контейнера)
- `docker container stats` - Показать ресурсы, потербляемые контейнерами
> [!note] Лайфхак
> Вместо полного id можно указать всего лишь первые несколько символов если они не совпадают с другим контейнером
## Port mapping

> [!info] Каждый контейнер имеет свой внутренний ip-адрес по умолчанию

`docker run -p 80:5000 [image name/id]` - Запустить контейнер и пробросить внешний порт `80` на внутренний `5000`
## Volume mapping

>[!info] Во время остановки контейнера все изменения в контейнере не сохраянются по умалочанию

Чтобы хранить данные, нам необходимо ==подключить внешнюю директорию или диск== к нашему контейнеру c помощью команды:
`docker run -v [outer path]:[internal path]`

Для того чтобы ==скопировать данные из контейнера на локальный хост==, необходимо воспользоваться командой:
`docker container cp [container name/id]:[internal path] [outher path]`
